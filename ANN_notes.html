Data

  • X: data-matrix

    • size n×m
    • n data-points
    • m features

  • Y: one-hot label matrix

    • size: n×k
    • n data-points
    • k classes

Model

The model has three components:

  • layers
  • weights
  • activation functions

The number of layers and their sizes determine the weights. Activation functions are independent of the layers and weights. With these components, we can define the model equations that compute the output given the input. In the case of neural networks, this is called the forward pass.

 

Layers

The model has L+1 layers. Layers are indexed using l.

  • l=0: input layer
  • 1≤l≤L−1: hidden layers
  • l=L: output layer
  • Sl: number of neurons in layer l

S0=m, the number of features for the problem. SL can take the following values:

  • 1, for a regression and binary classification problem
  • k, for a multi-class classification problem

 

Weights and biases

The weight matrix connecting layer l−1 and l is as follows:

  • Wl: weight matrix

    • size: Sl−1×Sl
    • Wlpq: weight connecting pth neuron in l−1 to qth neuron in l

  • bl: bias vector

    • size: Sl
    • blq: bias of the qth neuron in l

W0=I and b0=0, where I is the identity matrix and 0 is the zero-vector. There are no weights before the input layer. This convention is for consistency of notation when we come to the forward pass equations. The learnable weights and biases are for 1≤l≤L.

 

Activation functions

The type of the activation function depends on the layer in which it is used:

  • hidden layers
  • output layer

 

Hidden layers

g(z) will be used to refer to activation function. z is some real number.

 

  • Linear: g(z)=z

     

  • Sigmoid: g(z)=11+e−z

     

  • Tanh: g(z)=tanh⁡(z)

     

  • ReLU: g(z)={z,z>00,otherwise

 

When the argument to g is a matrix, the activation function will be applied element-wise.

 

Output layer

The activation function at the output layer depends on the type of problem being solved: regression or classification, and could be one of the following:

  • Linear: g(z)=z

    Here, z is a scalar activation at the output, that is, there is exactly one neuron in the output layer. This is used for regression problems.

     

  • Sigmoid: g(z)=11+e−z

    Here, z is a scalar activation at the output, that is, there is exactly one neuron in the output layer. This is used in binary classification problems.

     

  • Softmax: g(z)=[⋯ezi∑r=1kezr⋯]T

    Here, z is a vector of activations at the output layer, that is, there are k neurons in the output layer. This is used in multi-classification problems.

 

Activations

For any layer, there are three steps:

  • Accept input
  • Linearly combine the inputs
  • Apply the activation function

What comes out of a layer are called the activations at that layer. The linear combination of the inputs to a layer are called the pre-activations.

Zl and Al represents the matrix of pre-activations and activations respectively at layer l for 0≤l≤L. A0=X, the feature matrix. The activation matrix Al is of size n×Sl. Each row of the activation matrix corresponds to the activation vector for one of the data-points n data-points.

 

Forward pass

If there are k neurons in the output layer, the network can be seen as a function h:Rm→Rk. In the case of a multi-class classification problem, given a data-point x∈Rm, the network outputs a vector of probabilities.

h(x)=y^

If the feature matrix X of size n×m is passed as input, we get a matrix of probabilities, Y^ of size n×k. For a regression problem, k=1 and the output is interpreted as some real number. This input-output relationship can be computed in an iterative manner. This is termed as a forward pass.

If Al−1 represents the activation matrix at layer l−1, then the activations at layer l can be computed iteratively using the following pair of equations:

Zl=Al−1Wl+bl,1≤l≤LAl=g(Zl),1≤l≤L

Here, A0=X and W0=I and b0=0. The shapes of these matrices/vectors are as follows:

  • Al−1: n×Sl−1
  • Wl: Sl−1×Sl
  • bl: Sl
  • Zl: n×Sl
  • Al: n×Sl

Note that bl gets added to each row of the product Al−1Wl according to NumPy broadcasting rules. g is the hidden-layer activation function for 1≤l≤L−1 and the output-layer activation function for l=L. The final shape of the output activations at layer L is:

  • n×1 for regression and binary classification problems
  • n×k for a multi-class classification problem

According to our notation, AL=Y^. To understand more about the mathematical details behind the form of Zl refer to appendix item-1.

Loss

Regression

y is a vector of target label for n examples. y^ is the output of the network and corresponds to the predicted labels. Both vectors are of size n. The loss is our usual squared error:

L(y,y^)=12⋅(y^−y)T(y^−y)

Multi-class classification

Y is a matrix of target labels for n examples. Each row of this matrix is a one-hot vector. Y^ is a matrix of predicted probabilities. Both matrices are of size n×k. The categorical cross-entropy loss is given as follows:

L(Y,Y^)=−1nT(Y⊙log⁡Y^)1k

This equation can be understood as follows:

  • 1n and 1k are vectors of sizes n and k respectively.
  • If M is a matrix of size n×k, then 1nTM1k is the sum of all elements in the matrix.
  • ⊙ is the element-wise product.

Optimization

We need to compute the gradients with respect to the weights and the biases. For each weight matrix Wl, let us denote the gradient of the loss with respect to it as Wl(g). The algorithm to compute these gradients systematically is termed as the backpropagation. Just like we had a "forward pass" to compute the output given the input, we have a "backward pass" to compute the gradients from the loss.

Backward pass

We need to compute the gradients of the loss function with respect to the weights. The backward pass can be divided into two parts:

Hidden Layers

The weights at layer l, Wl(g), influences the loss via the matrix of activations, Al, at layer l. The equation that connects these two quantities is given below:

Zl=Al−1Wl+bl,1≤l≤LAl=g(Zl),1≤l≤L

Let us assume that we already have access to the gradient of the loss with respect to the activations at layer l. Let us call it Al(g). If we need the gradient of the loss with respect to the weights at layer l, then by the chain rule of differentiation, we need to compute the gradients with respect to the pre-activations at layer l. We shall use the following notation for the gradients at layer l:

  • Wl(g): gradient of the loss with respect to the weights
  • Zl(g): gradient of the loss with respect to the pre-activations
  • Al(g): gradient of the loss with respect to the activations

The rule is quite simple to state. It is just a product, the first one is element-wise, the second one is matrix multiplication:

Zl(g)=Al(g)⊙g′(Zl)Wl(g)=Al−1TZl(g)

To propagate this process to earlier layers, we also need Al−1(g). The expression for that is quite similar:

Al−1(g)=Zl(g)WlT

To understand the mathematical details, refer to appendix item-1.

Output layer

The only thing that remains is to compute the gradient of the loss with respect to the activations in the last layer. This depends on the form of the loss, which in turn depends on the type of problem being solved:

Regression

Recall that AL=y^. The activations at the last layer are the predicted labels.

AL(g)=y^−y

As the output-activation function is linear, computing the gradient of the activations and weights for the penultimate layer in the network is straightforward. In fact, ZL(g)=AL(g).

 

Multi-class classification

Recall that AL=Y^. The activations at the last layer are the predicted probabilities.

AL(g)=−Y⊙Y^⊙−1

This notation might be new. P⊙−1 is element-wise inverse. Pij⊙−1=1Pij. This inverse arises from differentiating the the log term in the loss function. As the output-activation function is softmax, computing the gradients of the pre-activations is less straightforward. Refer to appendix for a detailed derivation of the same. The expression for the gradients turns out to be very simple in the end:

ZL(g)=Y^−Y

Item-2

Chain rule for matrix product

Let P=QR, where P,Q and R are matrices of compatible dimensions. Let ϕ be some function of P. Let us assume that we have access to the gradient of ϕ with respect to P. Call this P(g). Then, we have the following equations:

Q(g)=P(g)RTR(g)=QTP(g)

This is nothing but the chain rule of differentiation in the presence of a matrix product.

 

Item-3

Gradient for softmax layer

In order to derive the gradient matrix at the final layer, it is easier to consider the simpler case of the gradient vector. Let z be the pre-activation vector at layer L of the network. After applying softmax function g on the pre-activation vector, we get a, the activation vector. The two vectors are related by the following equations:

a=g(z)=[⋯ezj∑r=1kezr⋯]

The gradient of the loss with respect to the activation vector is given by:

a(g)=∇aL

The tricky part is the gradient of the loss with respect to the pre-activations. Let us consider a single element in the pre-activation vector, zj. We are interested in the partial derivative ∂L∂zj. What this means is this: how much does the loss change when a small change is made to zj. It is not exactly this, but the ratio of these two quantities, but you get the idea.

zj isn't directly connected to the loss function. The elements of the activation vector ais intervene. In other words, the impact of zj on the loss is mediated by the ais. Changing zj has an impact on each of the ais because of the way softmax is defined. We already know how the loss is affected when the ajs are disturbed. This is captured by a(g). So much for the chain rule. We are now ready to state it:

∂L∂zj=∑i=1k∂L∂ai∂ai∂zj

The second term in the product is the ijth element of the Jacobian matrix Jz(a):

Jz(a)|ij=[∂ai∂zj]

The Jacobian matrix is a matrix of partial derivatives of one the components of the vector a with respect to z. With the introduction of the Jacobian matrix, we can now state the gradient of the loss with respect to the pre-activations in the following succinct manner:

z(g)=∇zL=Jz(a)Ta(g)

We can now turn our attention to computing the Jacobian:

Jz(a)|ij={−aiaji≠jaj−aj2i=j

This form of expressing the Jacobian element-wise is unwieldy for computation. Fortunately, this has a simple matrix-representation:

Jz(a)=diag(a)−aaT

Here, diag(a) is a diagonal matrix with the elements of the activation vector making up the diagonal. The second term on the RHS is called the outer-product. We can now plug the Jacobian into the earlier equation:

z(g)=Jz(a)Ta(g)=[diag(a)−aaT]a(g)=(a⊙a(g))−(aTa(g)a)=(a⊙a(g))−(a⊙a(g)1ka)

That might seem terribly complicated. Step-2 of the equation is just substituting the Jacobian. To understand step-3, try to think about the product of a diagonal matrix and a vector, and see how that transforms to element-wise product between two vectors. The final-step is another trick where we convert the dot product between two vectors into an element-wise product followed by multiplication by a vector of ones.

Why are all these transformations necessary? The moment we express them as element-wise products, we can almost effortlessly extend this formula to a matrix of activations:

ZL(g)=(AL⊙AL(g))−(AL⊙AL(g)1k×k)⊙AL

That again looks complicated. But if we stare at it for a while, the RHS is actually remarkably simple:

  • The first term is is simply the element-wise multiplication of the activation matrix and its corresponding gradient. Call this some matrix B.
  • The second term is just the row-wise sum of B followed by an element-wise multiplication with the activation-matrix.

These expressions are powerful because we can directly translate them into NumPy. For example, the NumPy equivalent of this equation would be:

xxxxxxxxxx

1
B = A_L * A_L_g
2
Z_l_g = B - np.sum(B, axis = 1) * A_l

The good news is that the expression for ZL(g) In the case of softmax with categorical cross-entropy loss can be further simplified. Recall the following results:

AL=Y^AL(g)=−Y⊙Y^⊙−1

The element-wise product of these two matrices is simply the matrix −Y! With that, the expression for ZL(g) becomes:

ZL(g)=Y^−Y
